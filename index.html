The Opportunity

The most-asked question is the least answered.

"Check upgrade eligibility" is the single biggest upgrade contact driver — 111,802 contacts in 15 months — and today effectively none of them are contained. Yet eligibility is a bounded, data-driven answer: the API exists, and the response tells us the route, the date, the settlement figure and the fee. It is exactly the kind of question an agent should answer in seconds.

AnswerEligibility, in-chat

The agent checks the customer's real position — eligible now, eligible from a date, settlement to pay, or Switch Up available — and says so plainly, with the figures.

ExplainOptions without jargon

Refresh unlinking, early upgrade fees, keeping your airtime while changing the phone — explained in plain English, with total costs before any commitment.

GuideTo checkout, never past it

The agent walks the customer through the web or app checkout step by step. It never places the order, takes payment or runs the credit check — the customer does.

Entry design this leads to: pass the customer's verbatim opening message into GECX at handoff. 75% of customers get a flying start; the mislabelled 11% are caught faster, because the utterance is exactly the evidence the agent needs to reroute — a confirmation gate would cost every genuine customer a turn and still leave the agent blind. The Journey

Four steps, one conversation.

The agent's backbone, whatever the customer's starting point. Arrive mid-journey — "I want the Pixel" — and it meets you there, checking the earlier steps as it goes.

STEP 01

Identify the line, check eligibility

Multi-line accounts pick which phone first — every API is keyed on the number. Then the real position: eligible now, a date, a settlement, or Switch Up.

Generative UI · A2UI

The agent stops describing and starts showing.

Comparing three phones in prose is a wall of text — worse out loud. With A2UI, the agent emits a JSON description of the interface it wants — and this page renders it with components we own, styled to the brand, safe by construction. The agent never generates code; it can only reference components from a catalogue the front end declares. Two client-side function tools carry the whole pattern: get_a2ui_catalog tells the agent what this page can render, use_a2ui_component renders one instance. It works in chat and in voice — a spoken "I've put your options on screen" while the card appears, and a tap on the card becomes the customer's next turn.
